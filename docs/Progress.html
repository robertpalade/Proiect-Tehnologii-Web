1. Web design

We started the project by creating the main elements such as navigation bar, the home page, and the pages where charts will take place. Also, at this step we made the charts( pie chart, bar chart, line chart and map chart) using some dummy data.

2. API

At this step we created a database that stores the information available in CSV format on the official site of the fleet.

3. Select boxes

There were created some select boxes for county, brand, national category and community category to generate the charts depending on the data which you choose from the select boxes.

4. Download buttons

At this step we implemented the download buttons for exporting data in three formats: CSV, SVG, WebP. For this step we made 2 functions for every chart, one for build and the other one for updating the chart every time the user selects something else from the select boxes.

5. Updated comboboxes

For each way of data visualisation we added one or two select boxes. When there are two select boxes, the second one updates depending on the first one.

6. Code Restructuring & Hardcoding Removal

For every page of data visualisation we had a different JavaScript file, so we created just one for every type of chart, and a single buildChart javascript file.

7. API update

We created a script which generates automatically the database with all the information.

8. Login and Register

At this step we added login and register pages, such that if somebody wants to enter the webpage he/ she needs to create an account firstly.

9. Added header as a separate file in all pages

I've created a new JavaScript file which contains the header because we have too many html pages with the same code.

10. Admin page

We have two types of users: normal user and administrator. The administrator has a new page from where he/ she can hide the statistics he/ she wants.

11. Responsiveness

The page can be see also on a table or smartphone.

12. Validate CSS & HTML

At this step we elimanated some warnings that we had with the CSS file and HTML pages.

13. Unit testing

We made some unit tests for API.
